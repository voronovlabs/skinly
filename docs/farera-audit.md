## Платформа

`cheerio`-fetcher подходит без Playwright. Есть XML-карта сайта
(`/sitemap.xml` → `/products1.xml`, `/feature_variants1.xml`).

### Польза для Skinly

**Комплементарный, но НЕ barcode-источник.** Сильные стороны: чистый INCI
(там где есть) — прямое топливо для compatibility-engine KB; структурированные
`Тип кожи / Линия / Средства / Страна`; качественные изображения и описания
профессиональных/аптечных марок, которых может не быть в Нацкаталоге.
Слабость: **нет EAN** → товары нельзя сматчить через сканер штрихкода. Матчинг
к существующему `Product` — по `brand + name` (fuzzy) либо как отдельный
name-searchable каталог.
